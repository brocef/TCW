As a user, after I have overridden a federated capability with [`tcw capabilities set`](tcw://C/capabilities/override-inherited), I drop that local override and re-inherit the upstream entry verbatim by running `tcw capabilities reset <path>` — no hand-deleting of the override folder. `reset` removes only my local override and never touches the upstream node. If the path has no local override it fails clearly: a standalone local capability tells me to use `remove` instead, and a path that already inherits verbatim tells me there is nothing to drop. `reset` reverts the whole override at once; to change a single inherited field back I use `set … --field K=<value>` instead.
