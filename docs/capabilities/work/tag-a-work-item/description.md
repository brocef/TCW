As a user, I classify a work item with one or more **tags** drawn from a set my project registers centrally. I register the valid tags with `tcw work tags add <tag>` (and `tcw work tags rm <tag>` / `tcw work tags list`), which maintains the registered set in the node's `tcw-config.yaml`; only registered tags may be applied. I apply tags with `tcw work new "<title>" --tag <tag>` (repeatable) or `tcw work edit <slug> --tag <tag> --untag <tag>`, and applying a tag that is not registered is refused with an error. Tags are shown by `tcw work show`, appear as a column in `tcw work list`, and I filter the board to items carrying a tag with `tcw work list --tag <tag>` (repeatable = match any). The same tags surface in the `tcw serve` web interface, where I can view and edit an item's tags against the registered set. `tcw validate` flags any item carrying a tag that is no longer registered.
