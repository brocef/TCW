# Upcoming

Developer changelog for the next version. Technical and precise; grouped by
category, with commit hash ranges so entries trace back to source.

## Fixed

- 989e2f0..HEAD — Import crash on Python 3.11–3.13 (`pyproject.toml` declares
  `requires-python = ">=3.11"`, but importing the package raised at class-
  definition time; only 3.14 worked, via PEP 649). Two eager-annotation
  landmines in `tcw/store/`: a builtin-shadowing `def list` on `TaxonomyStore`/
  `CapabilitiesStore` whose *subsequent* methods' `list[...]` annotations
  subscripted the method object (`'function' object is not subscriptable`), and
  `"TermDetail" | None`-style stringized forward-ref unions (`unsupported
  operand type(s) for |: 'str' and 'NoneType'`). Added `from __future__ import
  annotations` to `tcw/store/base.py` and `tcw/store/fs.py` to defer annotation
  evaluation. Verified: full import + suite green on real 3.11/3.12/3.13.

## Changed

- 989e2f0..HEAD — Renamed the builtin-shadowing `TaxonomyStore.list` /
  `CapabilitiesStore.list` (and their `FsTaxonomyStore`/`FsCapabilitiesStore`
  realizations) to `list_all`, updating all call sites (CLIs, `tcw/serve`,
  tests). Internal store-interface method only — the `tcw taxonomy list` /
  `tcw capabilities list` CLI subcommands are unaffected.
