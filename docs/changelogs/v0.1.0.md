# v0.1.0

Developer changelog for v0.1.0. Technical and precise; grouped by category,
with commit hash ranges so entries trace back to source.

<changes starting-hash="fed7d62" ending-hash="b03cb00">

Cross-node recursion — work Spec 2.

### Added
- **Node topology** (`tcw/store/fs.py`): `child_nodes`/`parent_node` + `_git_common_dir`
  — FS-adapter-local discovery of a node's nearest descendant/ancestor nodes
  (git work-tree + `docs/work/`). Own linked worktrees are excluded via an
  absolute `--git-common-dir` compare; nested independent repos are kept.
- **Recursion layer** (`tcw/work/recursion.py`, new): `reconcile(node_root,
  epic_slug)` scans this node + children for `initiative == epic_slug` and writes
  a deterministic, (node, slug)-keyed rollup into a managed `<!-- tcw:rollup -->`
  block in the epic's `content.md` (idempotent; read-only on `capabilities.yaml`,
  tolerant of malformed blobs). `delegate`/`escalate` write a dated request doc
  into a child/parent node's `inbox/` only (boundary-enforced).
- **`WorkItem` fields**: `initiative`, `type`, `worktree`, `branch` (all optional,
  back-compatible); surfaced via `FsWorkStore.get`.
- **Worktree plumbing** (`tcw/store/fs.py`): `git_commit` (scoped partial commit),
  `ensure_worktree_ignored`, `add_worktree`, `remove_worktree`, `WORKTREES_DIR`.
- **CLI** (`tcw/work/cli.py`): `nodes`, `reconcile [--commit]`, `delegate`,
  `escalate`; flags `new --epic/--initiative`, `edit --initiative`,
  `start --worktree`.
- **Tests** (`tests/test_recursion.py`, new): topology (incl. worktree-exclusion
  vs. nested-repo), reconcile (node-keying, idempotency, capability surfacing,
  malformed degradation, unknown-epic error), inbox channel (boundary, collision,
  root error), worktrees (item-in-worktree, clean merge-back, teardown).

### Changed
- `start --worktree` commits the `backlog→active` transition before
  `git worktree add` (a deliberate exception to the stage-only default — a
  worktree must branch from a commit holding the item at `active/`). `complete`
  is now a CLI wrapper that, post-integration, runs the unchanged
  `WorkStore.complete()` then best-effort tears the worktree down. The
  `WorkStore` ABC is unchanged — cross-node is a layer above it.
